# Security Policy

## Reporting a vulnerability

If you have found a security issue in this project, please report it privately by emailing sarma@sarmalinux.com. Do not open a public GitHub issue, and do not disclose the issue elsewhere until it has been addressed. Include a clear description, steps to reproduce, the commit SHA you tested against, and any proof-of-concept output that helps me confirm the problem quickly.

## Response policy

I respond to every disclosure within 7 days of receipt with an acknowledgement and an initial assessment. Confirmed issues are patched on `main` and released as a tagged version, and I will credit you in the release notes unless you ask me not to. Only the latest commit on `main` receives security fixes, so pin to a tagged release if you need a stable surface.
